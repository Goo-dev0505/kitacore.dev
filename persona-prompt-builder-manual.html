Persona Brand Prompt Builder
Manual

GPT-Image-2 で日本語ブランドポスターを一発生成するためのプロンプトビルダー。 このマニュアルでは、各フィールドの丁寧な書き方、チップ選びのコツ、 ChatGPTに投げた後の運用方法まで一通り解説します。

TARGET GPT-IMAGE-2 READ ≈ 10 MIN LEVEL BEGINNER → PRO
00. Overview

このツールで何ができるかWhat this tool is for

クリエイター情報・キャラクター設定・ブランドテーマをフォームで入力すると、 GPT-Image-2 用のポスター生成メタプロンプトが自動生成されます。 ChatGPT（画像生成ON）に貼り付けると、ブランド分析＋日本語見出し案＋英語プロンプト＋ポスター画像 が一気通貫で出力されます。

向いているユースケース

  • noteのプロフィール用ブランドポスター
  • マガジン・メンバーシップのカバー画像
  • 自己紹介スライドの中表紙
  • Xプロフィール添付画像、固定ポスト用画像
  • 個人事業・屋号のビジュアルアイデンティティ試作

向いていないユースケース

NOT FOR 企業の正式ロゴ制作、著作権が絡む二次創作、実在人物の肖像利用。 あくまで「自分のブランド」を立ち上げるための試作用途として使ってください。

ツール本体は左右2カラム構成。左が入力フォーム、右が生成プロンプトのリアルタイムプレビューです。

入力フォーム

上から順に：

  • 01. Creator Info — クリエイター情報

RIGHT / PREVIEW

上部のボタン群

ボタン役割使いどころ
サンプルを読み込む KITAcoreの見本データを全フィールドに一括投入 初回起動時に押して、書き換える形で使うと最速。

02. Field Guide

入力項目ガイドHow to fill each field

全15項目の「何を書くか」「書き方の例」「避けたいNGパターン」を一覧化。 迷ったら該当箇所だけ見てください。

01. Creator Info（クリエイター情報）

フィールド何を書く例とNG
creator_nameCreator Name* noteやXで使っている活動名。本名じゃなくてOK。ポスター内に描画される可能性あり。 OK： KITAcore / Goo / ktcrs
NG： 「クリエイター」などの一般名詞（特定できない）
brand_nameBrand Name* 活動の屋号・ブランド名。クリエイター名と同じでも可。ポスターのメイン見出しに入る。 OK： KITAcore Studio / 実験と観察
NG： 長すぎる（20字超）
main_urlMain URL* ChatGPTが読み込む最重要ソース。noteのトップページで十分。 OK： https://note.com/ktcrs1107
NG： URLの http:// を省く
profile_urlProfile URL 自己紹介記事・固定プロフィール。空欄可。 あると人物像の精度UP。なければ空欄でOK。
ref_urlReference Article 「このブランドを象徴する記事」を1本。スタイル判定の補助。 AI実験ログ系なら代表的な実験記事1本。プロフィール記事とは別物推奨。

02. Character（キャラクター情報）

ポスター中央に描かれる人物の設定。本人に寄せる／アバター／完全架空のペルソナ、どれもOK。

フィールド何を書く例とNG
char_genderGender 性別。プルダウンから選択。 male / female / androgynous / non-binary
char_ageAge Range 年代。幅のある表現で。 late 20s / early 30s / mid 30s ...
char_cultureCulture 国籍・文化圏。日本人はデフォルトでJapanese。 OK： Japanese / East Asian
NG： 空欄（文化圏が描画に効くので必ず記入）
char_roleRole 職業的イメージ。肩書より「雰囲気」優先。英語キーワード2〜3個。 OK： creative technologist, data storyteller, prompt engineer
NG： 正式な肩書だけ（例：Senior Engineer at XYZ）
char_appearanceAppearance 外見特徴。髪・メガネ・服装の方向性を英語で。 OK： short black hair, minimal round glasses, clean casual outfit in neutral tones
NG： 細かすぎる服飾指定（AIが迷う）
char_expressionExpression 表情・雰囲気。形容詞3つ程度で。 OK： calm, curious, slightly smiling
NG： happy だけ（情報不足）
char_posePose ポーズ・構図。身体のどの部分が写るかも含める。 OK： half-body, looking slightly off-camera, holding a notebook
NG： cool pose（抽象すぎ）

03. Brand System（ブランドテーマ・トーン・カラー）

チップクリックで追加。候補にない語は下の入力欄でEnter追加可能。 それぞれ 3〜6個 選ぶとバランスが一番出ます。

カテゴリ何を選ぶ選び方のコツ
Brand Themesthemes ブランドを象徴する視覚モチーフ・発信テーマ。 自分のnoteの記事タイトルを一覧して、頻出してる雰囲気を選ぶ。例：sunlight technology data storytelling
Brand Tonetones 読者に持ってほしい印象。 形容詞中心。例：intelligent calm editorial
Color Palettecolors ポスター全体の配色。 ベース1色＋アクセント1〜2色＋中間色2〜3色。
例：deep navy warm sunlight gold off-white
TIP チップ式の候補は全カテゴリで計50個以上用意してます。 迷ったら直感で「これかも」と思うやつを全部クリックして、後から減らすほうが早い。
03. Tips

使い方のコツPractical tips

実際に使って「精度が上がった」と感じた設計のコツを10個にまとめました。

  1. まずサンプルを読み込んで、置き換える

    空のフォームから書き始めるより、サンプル → 各フィールドを自分のデータに上書き するほうが圧倒的に早いし、抜け漏れが出ない。最初の1回は必ずサンプルから入るのを推奨。

  2. 3〜6個ルール

    テーマ・トーン・カラーはそれぞれ3〜6個が最適レンジ。 2個以下 → キャラクターが立たない。7個以上 → 散らかって焦点ボケる。

    迷ったらまず6個選んで、プレビューを見ながら「これは違うな」を削る方式が安全。

  3. カラーは「ベース＋アクセント＋中間」の3層で

    単なる色の羅列より、役割を意識すると一貫性が出る。

    例：ベースdeep navy + アクセントwarm sunlight gold + 中間off-white muted sage charcoal black

  4. Appearance は「3要素＋雰囲気1語」で止める

    髪・メガネ・服装 の3要素と、全体の雰囲気を1語。 書きすぎるとAIが優先順位を迷って破綻する。

    例：short black hair, minimal round glasses, clean casual outfit in neutral tones, quietly intellectual vibe

  5. 日本語と英語の使い分けを固定する

    Creator Name / Brand Name は日本語のまま（ポスターに直接描画される）。 Character系・Brand系は英語（描画指示として渡される）。

    GPT-Image-2 は日本語描画が得意だが、「どう描くかの指示」は英語のほうが精度が出ます。

  6. ref_url（参考記事）は「自分らしさMAX」の1本を

    入れるなら、自分のスタイルを最もよく表している記事を1本だけ。 プロフィール記事とは別物として扱ったほうが、人格とスタイルを別レイヤーで抽出してくれる。

  7. プレビューを見ながら調整する

    右側のプレビューは入力と同時にリアルタイム更新。 「入れてみる → プレビューの該当箇所を確認 → 違和感あれば直す」のループを意識すると、 ChatGPTに投げる前に完成度を上げられる。

  8. ブラウザ保存は「作業区切り」で押す

    各STEPを終えるごと、または30分ごとに「ブラウザに保存」。 localStorage なのでタブを閉じても残るが、シークレットモードでは消えるので注意。

  9. 複数案を作るときは .md でダウンロードする

    パターンA・Bを試したいときは、各パターンを .md でダウンロードして保存。 ブラウザ保存は1セットしか持てないので、複数案管理はファイル保存がおすすめ。

  10. 一発で決まらなくて当たり前と思う

    画像生成は確率的。1回目で気に入らなくても、同じプロンプトで2〜3回再生成するのが普通。 「プロンプト修正は3回目以降」くらいの感覚で。

04. Workflow

ChatGPTでの運用How to actually generate the poster

ビルダーから出たプロンプトを、実際にどう使うか。2つのルートがあります。

ルートA：ChatGPTでワンストップ（推奨）

ChatGPTを開く

画像生成機能が使えるモデル（GPT-5など）を選択し、画像生成ONの状態にする。

プロンプト全文を貼り付け

ビルダーでCopyしたメタプロンプトをまるごと入力欄に貼る。

Copyボタンで全文取得済みなので、Ctrl+V（⌘+V）1発でOK。

Enterで送信 → 分析とプロンプト生成を待つ

ブランド分析（日本語）→ ポスター構成仕様（日本語）→ GPT-Image-2用プロンプト（英語）の3段が返る。

画像生成をリクエスト

「このプロンプトで画像を生成してください」と追加で指示。 縦長ポスター希望なら 「縦長ポスター（A3相当）で生成して」 と明示。

出力の調整

気に入らない要素があれば、「人物の表情をもう少し柔らかく」「左上の見出しを大きく」など日本語で追加指示。

ルートB：API直接（開発者向け）

OpenAI の Images API で gpt-image-2 モデルを使う場合。 メタプロンプト出力の「# 3. GPT-Image-2 用プロンプト」の英語部分だけを prompt パラメータに渡します。

POST https://api.openai.com/v1/images/generations { "model": "gpt-image-2", "prompt": "<3段目の英語プロンプト全文>", "size": "1024x1536", "n": 1 }
RECOMMENDED SIZE 縦長ポスター：1024x1536 ／ 横長：1536x1024 ／ 正方形：1024x1024

リトライのコツ

同じプロンプトで3回まで試す

画像生成は確率的なので、1回目が微妙でもプロンプトを変えずに再生成すると別解が出ます。 3回試して改善が見られない場合だけ、プロンプトを修正。

修正するなら1箇所ずつ

複数箇所を同時に変えると、どの修正が効いたかわからなくなる。 色 → 表情 → ポーズ、のように1変数ずつ変えるのが科学的。

05. Troubleshooting

トラブルシュートWhen things go wrong

よくある困りごとと、対処法をまとめました。

症状原因対処
プレビューが更新されない 必須3項目（Creator Name / Brand Name / Main URL）が未入力 3つとも埋めると生成が始まる。STATUSが "WAITING" → "READY" に変わればOK。
Copyボタンで何もコピーされない ブラウザのクリップボード権限が無効／HTTPS以外で開いてる ローカルファイル（file://）では動作する。企業PCで制限されてる場合は .md ダウンロードで代用。
チップをクリックしても追加されない すでに選択済みなのでトグルオフされている 「Suggestions」下のチップに色がついてるものが選択済み。上の選択エリアで確認。
ポスター画像の日本語が崩れる GPT-Image-2 以外のモデルで生成している／日本語文字列が長すぎる モデルが gpt-image-2 になってるか確認。
ブランド名・見出しが長すぎる場合は短縮（15字以内推奨）。
人物の顔が希望と違う Appearance が抽象的 or 詳細すぎる 「3要素＋雰囲気1語」のルールに戻す。書きすぎを減らす。
ブラウザ再起動後、保存データが消えた シークレットモード／ブラウザ設定でlocalStorage無効化 通常モードで使う。重要なセットは .md ダウンロードで保険を。
情報パネル（CORE OS等）の内容が空っぽ ChatGPTの分析段階で情報が足りていない ChatGPTに「CORE OSパネルの中身を具体化してください」と追加指示。URLが有効でも情報が少ない場合は、note記事を3本ほど指定して再分析依頼。
06. FAQ

よくある質問Frequently asked questions

Midjourneyでも使えますか？▶
このツールはGPT-Image-2に最適化しています。Midjourney / Imagen / Nano Banana でも 意味は通じますが、日本語タイポグラフィの描画精度は大きく落ちます。 他モデルで使う場合は、日本語部分を英訳するか、Photoshop等で文字を合成してください。
参考記事URL（ref_url）は必須？▶
省略可です。入れると精度が上がりますが、なくても Main URL があれば分析できます。 迷ったら空欄でOK。
1つのブランドで何パターン作れば足りますか？▶
カラーパレット違い・トーン違いで 3〜5パターン作って比較するのがおすすめ。 1パターンだけだと「本当にコレでいいのか」の判断ができないので、必ず比較できる複数案を。 それぞれ .md でダウンロードしておけば後から再利用可。
キャラクターは自分の姿を反映すべき？▶
どちらでもOK。本人に寄せる／アバター的にする／完全に架空のペルソナにする、 いずれも成立します。note・Xのアイコンと整合させたい場合は、アイコンの特徴を Appearance欄にそのまま書くと連続性が出ます。
生成した画像の著作権は？▶
OpenAI の利用規約に準じます（2026年4月時点では、ChatGPT / Images APIで生成した画像は 商用含め利用可能）。最新の規約は OpenAI の利用規約ページを確認してください。 ただし、プロンプトに著作権がある既存キャラクター名を含めた場合は別途注意が必要です。
企業の広告やロゴに使っていい？▶
このツールは個人のブランディング試作を想定しています。 企業の正式ロゴや、商用広告のメインビジュアルには、プロのデザイナーに 依頼するのが筋。試作・ラフ提案・社内プレゼン用ならこのツールで十分です。
データはどこに保存される？▶
「ブラウザに保存」を押したときだけ、このブラウザのlocalStorageに保存されます。 サーバーには一切送信されません。別のブラウザ・別PCには引き継がれないので、 重要なセットは生成プロンプト(.md)をダウンロードして残すのが確実です。
オフラインで使える？▶
ツール本体（HTML）はGoogle Fonts読み込み以外はオフラインで動きます。 ただしChatGPTへの貼り付け以降はインターネット接続が必要です。
このツール自体を改変していい？▶
自由に改変・再配布OKです。KITAcoreクレジットを残してもらえると嬉しいです。 自分のワークフロー用にフィールド追加・チップ候補差し替えなど、ご自由にどうぞ。
