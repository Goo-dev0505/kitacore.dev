FIELD COMMAND Quick Start 03

5 Steps to
First Name

まず5ステップで最初のネームを作ろう。V5・V6 共通の基本フロー。

1
HTML ファイルを開く
ダブルクリックまたはブラウザにドロップ。インストール不要。V6はiPadのSafariで「ホーム画面に追加」するとフルスクリーン動作。
2
コマを分割する
白いコマを右クリック（PC）またはロングプレス（タブレット）→「左右に分割」などを選択。または右上の ⋮ ボタンをタップ。
3
セリフを入力する
コマをシングルタップ / クリック → テキストモーダルが開く。縦書き青で表示される。
4
下絵を描く
コマをダブルタップ / ダブルクリック → 描画モーダルが開く。V6ではApple Pencilの筆圧が線幅に反映。
5
保存 / 書き出し
サイドバー「↓ 現在のページをPNG」で書き出し。「💾 保存」でJSONに保存すると次回から続けられる。
⚡ AI使うと爆速： 「AI でネーム生成」からnote記事を貼るだけ。コマ割り・セリフ・プロンプトが全自動生成される。微調整するだけでネーム完成。

UNIT SPEC Pen Input System 06

Apple
Pencil

V6専用機能。Pointer Events APIで筆圧・パームリジェクションを完全実装。V5では非対応。

筆圧 → 線幅
Apple Pencil (全世代) / Samsung S Pen / Surface Pen に自動対応。
線幅 = 基本太さ × (0.3 + 筆圧 × 1.4)
PRESSURE
—
PALM REJECTION
手のひら検出
ペン検出後700msはタッチを完全ブロック。iPad上で手のひらを置きながらApple Pencilで描画可能。サイドバーのトグルでON/OFF可能。
HIGH FIDELITY
高精細描画
devicePixelRatio (最大3×)でRetina対応。getCoalescedEvents()で240Hzの高サンプリングレートも処理し、素早い線でも滑らか。
DRAW UNDO
描画内 Undo ↶
Draw Modal内に独立した Undoスタック（最大30ステップ）を搭載。間違えたらすぐ↶ボタンで戻せる。
✏️ Apple Pencil 第2世代： ダブルタップジェスチャーはiPadOS側の機能のためツール内切り替えには使えない。描画モーダルのツールバーボタンで「ペン」「消しゴム」を切り替えて。

+ + 12
INTEL Classified Q&A 12

FAQ

よくある質問と対処法。

V5で作ったJSONをV6で開けますか？▼
開けます。「📂 読込」からJSONファイルを選択すると、V5で作ったコマ割り・テキスト・下絵をそのまま引き継いで作業を継続できます。V6固有データ（ズーム値等）はデフォルト設定になります。
iPadでApple Pencilの筆圧が効かない▼
1. V6ファイルを使っているか確認。2. サイドバーの「筆圧対応 ✦」チェックがONか確認。3. Safariを使用（Chrome for iPadはPointer Eventsが制限される場合あり）。4. ページを再読み込みして再試行。
ロングプレスしてもメニューが出ない▼
コマ内の白い空白部分（テキストブロックや⋮ボタンの上ではない場所）を500ms押し続けてください。⋮ボタンをタップする方が確実です。
ダブルタップしてもテキストモーダルが開く▼
ダブルタップの2回目が280ms以内に届いていない可能性があります。より素早く2回タップしてみてください。最初のタップでモーダルが開いた場合は一度閉じてから再試行。
AI生成でエラーになる▼
原因：1. テキストが50文字未満。2. ネットワークエラー（少し待って再試行）。3. ファイルを直接開いている場合、一部ブラウザでfetch APIが制限される（ローカルサーバー経由で開くと解決）。
コマの削除がグレーアウトで選べない▼
ページ全体が1つのコマ（未分割）の場合、そのコマは削除できません。まず他のコマに分割してから削除してください。
