Why This Document Exists

AIは、知らない言葉を引き出せない

「売上が上がらない、どうしたら？」と聞いた答えと、「JTBDフレームで顧客の課題を分析して」と聞いた答えは、質が全然違う。AIはすでに答えを持ってる。

01 — WHAT

マーケティングとは何か

よくある誤解から入る

「マーケティング＝広告」やと思ってる人、今でも多い。広告はマーケの一手段であって、本体やない。それどころか、マーケを正しく理解すると「広告しなくてもいいビジネス」が設計できるとわかってくる。

同様に「マーケ＝SNS運用」「マーケ＝SEO対策」という誤解もある。これらは実行レイヤーの施策であって、マーケティング全体の中の一部分にすぎへん。

定義：3人の言葉で整理する

人物定義キーワード
Kotler（1967）個人や組織が価値を創造・伝達・提供し、顧客との関係を通じて組織と利害関係者に利益をもたらすプロセス価値・交換・関係
Levitt（1960）顧客の問題を解決することを事業の起点に置く考え方。製品ではなく顧客の「目的」を売る顧客志向・目的
Drucker（1954）マーケティングの目的は販売を不要にすること。顧客を深く理解すれば、製品は自然と売れる理解・需要創造

3つに共通するのは「顧客の価値を中心に置く」という視点。プロダクトやサービスを売ることが目的やなく、顧客が得る価値を設計することがマーケの本質や。

AI Unlock — 定義
この言葉を知ると引き出せること
「マーケティング」という言葉を知るだけでは弱い。KotlerやLevitt、Druckerという固有名を添えると、AIは「どの視点から答えるべきか」を理解して、格段に精度の高い分析を返してくる。
Prompt Example私のnoteをKotlerとDruckerの視点でそれぞれ定義してください。Kotlerなら「価値・交換・関係」の観点で、Druckerなら「販売を不要にする顧客理解」の観点で教えてください。

1行定義

Core Definition

マーケティングとは、「誰に・何を・どう届けるか」を設計し、価値交換を継続的に成立させるための思考と実行の体系。広告もSEOもその一手段に過ぎない。

なぜ「体系」と呼ぶのか

マーケが「体系」なのは、個別施策（SEO、SNS、広告）を単独で打つんじゃなく、調査→戦略→設計→実行→測定→改善という一連の意思決定プロセスとして動かすからや。どの施策をどのタイミングで使うか判断するには、全体の地図が要る。それを提供するのがマーケ理論の役割。

02 — WHY

なぜ生まれ、どう進化したか

歴史は「課題×時代」で読む

マーケ理論は「その時代の課題に対する答え」として登場してきた。だから時代背景と一緒に読まんと、なぜその理論が生まれたかがわからへん。

1900s–1950s
生産優位の時代
大量生産が可能になり「良いものを作れば売れる」時代。マーケはまだ「流通・販売の技術」として位置づけられてた。Fordの大量生産体制がこの時代の象徴。
1960s
顧客志向へのパラダイムシフト
Levitt（1960）が「マーケティング近視眼」を提唱。鉄道会社が「輸送業でなく鉄道業」と定義して衰退した例を挙げ、顧客の目的から逆算して事業を定義する重要性を説いた。Kotlerが初めてマーケを体系化したのもこの時代。
1980s
競争戦略の時代
Porter（1980）が「競争の戦略」を発表。5 Forcesモデルで業界構造を分析し、コストリーダーシップ・差別化・集中という3つの基本戦略を提唱。「どこで戦うか」というポジショニングの概念が重要になった。
1990s
ブランドと情緒価値の時代
Aakerがブランドエクイティ理論を確立。製品の機能だけでなく、ブランドが持つ意味・信頼・ロイヤルティが資産になるという考え方が広まった。差別化が難しくなる中で「誰が作ったか」が価値を持つようになってきた時代。
2000s–2010s前半
デジタルと計測の革命
インターネットが普及し、マーケが初めて定量的に計測可能になった。Google Analytics・SEO・コンテンツマーケが台頭。HubSpot（Halligan）がインバウンドマーケティングを体系化。「探されるマーケ」という発想の転換が起きた。
2011–2015
リーンとグロースの時代
Ries（2011）が「リーン・スタートアップ」を出版。MVP・Build-Measure-Learnループが普及。McClureのAARRRフレームと合わせて「仮説検証→高速改善」の思想がスタートアップから広がった。不確実性の高い市場でどう動くかの方法論が確立された。
2016–現在
AI・コミュニティ・個人の時代
プラットフォームの発達で個人がブランドを持てるようになった。Kotler（2021）はMarketing 5.0でAI・人間共存のマーケを論じる。コミュニティ・ファン・パーミッションが新しい軸として浮上。1対1から1対Nの価値共創に移行している。
読むポイント 各時代の理論は「前の時代の限界への回答」として出てきた。コトラーの体系化は生産優位時代の終焉、リーンはデジタル計測時代の「それでも外れ仮説を減らせない」問題への回答や。だから理論同士は対立してるんやなく、問題が違う。
03 — HOW

現在の地図（5層 MECEフレーム）

なぜ5層で切るか

マーケの概念を「SEO vs SNS」「古典 vs デジタル」で整理しようとすると必ず非MECEになる。なぜなら切り口が揃ってないから。

5層モデルは「意思決定の順序」で切ってる。何かを決めるとき、必ずこの順番でしか動けへん。だからMECEになる。判断の順序が異なる層同士は自然と被らへん。

5層マップ

L0 目的設定 なぜやるか
Mission / Vision Goal North Star Metric Golden Circle

事業・活動の存在理由を定義する。ここが揺れると全層がブレる。すべての判断の上位基準。Sinekの「Why→How→What」はこの層の核心を突いてる——Whyを持たない人は、何を売るかしか語れない。

L1 環境・顧客理解 どこにいるか
JTBD Persona 3C SWOT PEST 5 Forces TAM/SAM/SOM Customer Journey

顧客・市場・競合・自社の事実を把握する。仮説の材料になる。リーンの「顧客開発」もこの層。

L2 戦略 どこで・誰に・どう勝つか
STP Ansoff Matrix Porter 3 Generic Blue Ocean PMF

L1の事実から「どこで勝負するか」を決める。ここがずれると施策がいくら頑張っても効かへん。

L3 戦術設計 何を組むか
4P 4C Brand Equity Price Strategy Distribution

Product / Price / Place / Promotion の組み合わせを設計する。コトラーが体系化した中核。

L4 実行・届ける どのチャネルで動かすか
SEO SNS Content Paid AISAS Growth Loops Community

L2〜L3の決定を実際に動かすフェーズ。「SEOをやる」はここ。チャネルは時代で変わる。

L5 測定・最適化 効いてるか・学べてるか
AARRR LTV / CAC Cohort Analysis A/B Test KPI Tree

結果を数値で読んで、L1〜L4に学習を戻す。ここがループしてないと「やりっぱなし」になる。

層の判定基準

「この概念はどの層に入るか」で迷ったときの判定基準はシンプルや。「この概念が答えようとしてる問いはどれか」で分ければいい。

概念答えてる問い正しい層
コンテンツマーケどのチャネルで届けるかL4
ブランディングどう勝つか（L2）＋どう実装するか（L4）L2+L4 複合
LTV / CAC経済的に持続するかL5
JTBD顧客が何を片付けたいかL1
CRM継続顧客を測定・実行するL5+L4 複合
インフルエンサーマーケどのチャネルで届けるかL4
AI Unlock — 5層フレーム
この言葉を知ると引き出せること
「L0〜L5」「L1（顧客理解）」「L3（戦術）」という層の名前をAIに渡すと、「あなたは今どの層が弱いか」を5層で診断してもらえる。フレーム名なしだと「いろいろやってみましょう」という答えが返ってくるが、フレームを指定すると構造的な分析に変わる。
Prompt Example私のnoteを5層フレーム（L0目的設定・L1顧客理解・L2戦略・L3戦術設計・L4実行・L5測定）で診断してください。現状の強い層・弱い層を整理し、次に手をつけるべき層を1つ教えてください。
04 — WHO

主要流派と思想パッケージ

流派は「横軸」として理解する

流派（コトラー、リーン、グロースハックなど）は5層の縦軸を横断する「アプローチの束」や。各流派が「どの層を重視するか」「どんな哲学で貫くか」が違うだけで、どれも5層のどこかにマッピングできる。

Kotler系（深化型）

既知市場で勝つ

  • PMF以降が本領
  • L2〜L3が中核
  • STP・4Pが主要ツール
  • 既存市場の前提
  • 体系的・演繹的
Lean系（探索型）

未知市場を当てる

  • PMF到達前が本領
  • L1〜L2が中核
  • MVP・BMLループが主要ツール
  • 不確実性の前提
  • 仮説検証・帰納的
Key Insight コトラーとリーンは「対立」してるんやなく、時間軸上の前後関係や。PMF（Product-Market Fit）を境に使う道具が切り替わる。リーンでPMFを見つけ、コトラーでスケールする、が正しい順序。

主要流派マッピング

Kotler / Levitt
1960s–現在 · 伝統的マーケ
L2 L3
STP・4Pを体系化したマーケの「骨格を作った人」。顧客志向・価値交換が中心思想。Marketing 1.0→5.0まで時代対応しながら進化し続けてる。
Porter
1980s · 競争戦略派
L1 L2
5 Forces・3つの基本戦略・バリューチェーンで業界構造と競合優位を分析。「どこで戦うか」を決めるための道具。
Drucker
1950s–2000s · 経営管理論
L0 L1
「マーケの目的は販売を不要にすること」「顧客の創造」が有名な発言。目的層（L0）から顧客理解（L1）を貫く哲学的な立場。
Christensen
1990s–2010s · JTBD/破壊的革新
L1
Jobs-to-be-Done理論と破壊的イノベーション論。顧客が「片付けたいJob」を軸に製品を定義する。Levittの「穴を買う」の現代版。
Ries / Blank
2011s · Lean Startup
L1 L2
Build-Measure-Learnループ・MVP・Customer Developmentが主要概念。不確実性の高い市場での仮説検証方法論。PMF前の必修フレーム。
McClure / Ellis
2007–2010s · Growth Hacking
L4 L5
AARRRファネル（獲得・活性・継続・収益・紹介）を定義。PMF後のスケール戦略。North Star Metricで組織の測定軸を統一する。
Aaker / Keller
1990s · ブランド論
L2 L4
ブランドエクイティ・CBBEモデルを確立。ブランドが持つ認知・連想・ロイヤルティを「資産」として管理する考え方。差別化困難な市場での武器。
Godin / Kawasaki
2000s–現在 · コミュニティ/ファン
L4 L5
Tribes（部族）・Permission Marketing・Purple Cow。規模より「ファンの熱量」を優先する発想。個人クリエイター・note運用に最も直接的に刺さる流派。
ワイの推し
Simon Sinek
2009– · Start With Why / ゴールデンサークル
L0
Why（なぜやるか）→ How（どうやるか）→ What（何をするか）の順で語れ、という理論。多くの人はWhatから話す。でも人が動くのはWhyに共鳴するとき。TEDトーク「How great leaders inspire action」で有名。ワイがマーケを考えるとき、いつもここに戻ってくる。

流派の選び方

「どの流派が正解か」という問いは間違ってる。自分が今どのフェーズにいるかで使う流派が変わる。

フェーズ使う流派重視する層
PMF前（何が当たるか不明）Lean / JTBD / ChristensenL1 → L2
PMF発見〜検証中Lean + Growth HackingL2 → L4 → L5
PMF後・スケールKotler / Porter / AakerL2〜L5 総動員
個人・小規模クリエイターGodin / JTBD + LeanL1 + L4 + L5

05 — APPLY

noteに適用する

noteを5層で診断する

5層マップを使ってnote運用を診断すると「自分がどの層に強く・どの層が空白か」が見えてくる。施策ベースで考えてると気づかへんギャップが浮上する。

L0 — 目的
要確認

「なぜnoteを書くか」が言語化されてるか。収益化・認知拡大・思考整理・ブランド構築、どれを軸に置くかで全施策が変わる。

L1 — 顧客理解
部分的に実施

「スキをくれる人は誰か」「その人がどんなJobを持ってるか」のJTBD分析。スキ・ビュー・コメントのデータがL1の材料になる。

L2 — 戦略
未整備が多い

「どの読者層に・どのポジションで・どのテーマで勝つか」のSTP設計。ここが曖昧だと記事が散乱して読者層が育たへん。

L3 — 戦術設計
実践中

記事の価格設計（無料/有料/マガジン）・シリーズ構成・タイトル設計が4Pに相当。価格設計はここ。

L4 — 実行
強い

記事量産・SNS連携・ハッシュタグ・タイトル最適化。ツールを作る/購入で実行できる層。Growth Loopsの設計（シリーズ化→ファン化→紹介）を組み込むと加速する。

L5 — 測定・最適化
強い

スキ率・初速・遅延スキ・資産スコアの追跡。note-stats-trackerなどと組み合わせると自動収集化。この層が強いのはnoteクリエイターとしてかなりレアな強み。

noteに最も直接効く流派

推奨フレームの使い分け
JTBD
→
読者の「片付けたいJob」を記事テーマ設定に使う。「なぜこの記事を読むのか」の仮説が立てやすくなる。
Lean Loop
→
記事を「MVP」として仮説検証。スキ率が当たり判定。ピボット判断にもつながる。
STP（簡易）
→
L5データを使って「どの読者が・どの記事に・なぜ反応してるか」を逆算してSTPを定義する。
Godin / Tribe
→
メンバーシップ・マガジン設計の哲学に使う。「1000人のファン」が資産になる発想。

整理から次のアクションへ

5層診断で見えてくる「今やるべきこと」の優先順位はシンプルや。L5が強いうちに、L1（JTBD）とL2（STP）を言語化するのが最短経路。

なぜなら、L5のデータ（スキ率・資産スコア）が最高の顧客理解ツールとして機能してるから。「当たった記事に共通する読者像」を抽出して、JTBDに変換して、STPで整理する。このフローで地図が完成する。

Next Action

L5データを使って「当たり記事の読者像をJTBDで言語化」→「簡易STPを設計」。この2ステップで、全施策に一貫性が生まれる。
