03b — Candlestick

ローソク足
チャート

始値・高値・安値・終値（OHLC）を 1 本のローソクで表現。上ヒゲ・下ヒゲ・陽線・陰線で値動きの強さが一目でわかる。

▶ PREVIEW — ローソク足
cell_03b_candlestick.py
# @title 03b — ローソク足チャート ──────────────────── df = yf.Ticker(ticker_symbol).history(period=period, interval=interval).reset_index() fig = go.Figure(data=[go.Candlestick( x=df["Date"], open=df["Open"], high=df["High"], low=df["Low"], close=df["Close"] )]) fig.update_layout( title=f"{ticker_symbol} Candlestick ({period})", xaxis_rangeslider_visible=False # 下部スライダーを非表示 ) fig.show()
要素意味
陽線（緑）始値より終値が高い。その日は上昇した。
陰線（赤）始値より終値が低い。その日は下落した。
上ヒゲ日中の最高値。ここから売りが入った。
下ヒゲ日中の最安値。ここから買いが入った。

03c — Moving Average

移動平均
（SMA）重ね描き

終値を平滑化してトレンドを明確にする。2 本の線が交差するポイントで売買シグナルを読む。

cell_03c_moving_average.py
# @title 03c — 移動平均（SMA）を重ね描き ──────────── window = 25 # SMA の期間（日）。国内では 25日・75日 が一般的 df = yf.Ticker(ticker_symbol).history(period=period, interval=interval).reset_index() df["SMA"] = df["Close"].rolling(window=window).mean() # 移動平均を計算 fig = go.Figure() fig.add_trace(go.Scatter(x=df["Date"], y=df["Close"], name="Close", mode="lines")) fig.add_trace(go.Scatter(x=df["Date"], y=df["SMA"], name=f"SMA({window})", mode="lines")) fig.update_layout(title=f"{ticker_symbol} Close + SMA({window})") fig.show()
読み方
ゴールデンクロス：短期線が長期線を上抜け → 上昇転換のシグナル
デッドクロス：短期線が長期線を下抜け → 下落転換のシグナル

03d — RSI

RSI
（相対力指数）

過熱感を 0〜100 のスケールで数値化。終値と 2 軸で並べて表示する。

▶ PREVIEW — RSI(14)
cell_03d_rsi.py
# @title 03d — RSI（終値と 2 軸表示） ────────────── window = 14 # RSI の標準は 14 日 df = yf.Ticker(ticker_symbol).history(period=period, interval=interval).reset_index() delta = df["Close"].diff() gain = delta.where(delta > 0, 0).rolling(window=window).mean() loss = -delta.where(delta < 0, 0).rolling(window=window).mean() df["RSI"] = 100 - (100 / (1 + gain / loss)) fig = go.Figure() fig.add_trace(go.Scatter(x=df["Date"], y=df["Close"], name="終値", yaxis="y1")) fig.add_trace(go.Scatter(x=df["Date"], y=df["RSI"], name="RSI", yaxis="y2")) fig.update_layout( title=f"{ticker_symbol} 終値 + RSI({window})", yaxis =dict(title="株価 (円)"), yaxis2=dict(title="RSI", overlaying="x", side="right", range=[0, 100]) ) fig.show()
ゾーン判断
RSI > 70買われ過ぎ → 反落に注意
RSI 30〜70中立ゾーン → トレンドに従う
RSI < 30売られ過ぎ → 反発に注意（買い検討）

03e — Horizontal Line

水平線を
引くだけ

「過去の反転ポイントを見つけて横線を引く」。誰が引いてもほぼ同じになるから機能する。Plotly なら add_hline() で一発。

▶ PREVIEW — 水平線（支持線・抵抗線）
cell_03e_horizontal_lines.py
# @title 03e — 水平線（支持線・抵抗線）を引く ──────── # 過去の反転ポイントを見つけて横線を引くだけ ticker_symbol = "4755.T" # 楽天G period = "1y" interval = "1d" # キリバン（抵抗線・支持線）：キリがよいほど強いバリアになる levels = { 1000: ("1000円 — 最強バリア", "orange", "dash"), 900: ("900円 — 支持線", "royalblue", "dot"), 800: ("800円 — 次の支持線","royalblue", "dot"), } df = yf.Ticker(ticker_symbol).history(period=period, interval=interval).reset_index() fig = go.Figure() fig.add_trace(go.Scatter(x=df["Date"], y=df["Close"], name="Close", mode="lines")) for price, (label, color, dash) in levels.items(): fig.add_hline(y=price, line_dash=dash, line_color=color, line_width=1.5, annotation_text=label, annotation_position="right") fig.update_layout(title=f"{ticker_symbol} — 水平線（支持線・抵抗線）") fig.show()
ポイント内容
バリアの強さ1000円 > 500円 > 900円 > 800円。キリがよいほど強い
買いポイントバリア手前（例: 1000円バリアなら 920〜940円付近）でエントリー
利確ポイント上のバリアに達したら即利確。欲張らないのが重要
騙しに注意バリア上抜けに見えて急落することもある → 損切りを必ずセット
なぜ水平線がおすすめか
トレンドラインは引き方が人によってバラバラ。水平線は「誰が引いてもほぼ同じ」になる。テクニカル分析は多数決ゲームなので、みんなが同じ線を見ているほど機能する。

# @title 04a — 複数銘柄を重ね描き ───────────────── ticker_list = "9432.T,7203.T" # カンマ区切りで 10 社程度まで period = "6mo" tickers = [t.strip() for t in ticker_list.split(",")] df = yf.download(tickers, period=period)["Close"] df = df.reset_index().melt(id_vars=["Date"], var_name="Ticker", value_name="Close") fig = px.line(df, x="Date", y="Close", color="Ticker", title=f"Stock Comparison ({period})") fig.show()

# @title 04b — 正規化比較（初日 = 100） ──────────── # 株価の絶対値が違いすぎる場合に伸び率で比較する df = yf.download(tickers, period=period)["Close"] df = df / df.iloc[0] * 100 # 初日を 100 に正規化 df = df.reset_index().melt(id_vars=["Date"], var_name="Ticker", value_name="Index(%)") fig = px.line(df, x="Date", y="Index(%)", color="Ticker", title="Growth Comparison (Base = 100)") fig.show()

13
FAQ

よくある質問

Qデータが取得できない / エラーが出る
まず確認するのは ① 銘柄コード（日本株は 7203.T のように末尾に .T）、② インターネット接続、③ Yahoo Finance 側の一時的な制限。1分待って再実行するだけで直ることが多い。
Q短期足（1分足・5分足）を取りたい
Yahoo Finance 側の制限で短期足は取得できないことが多い。まず interval="1d"（日足）で試す。高頻度データが必要な場合は Alpha Vantage や Quandl（API キー必要）を検討するとよい。
QMACD・ボリンジャーバンドも実装したい
pandas_ta ライブラリを使うと多くの指標が 1 行で計算できる。!pip install pandas_ta でインストール後、df.ta.macd() や df.ta.bbands() を呼ぶだけ。
Q米国株・ETF・為替も分析できる？
できる。米国株は AAPL、ETF は SPY、為替は USDJPY=X のようにコードを指定するだけ。サフィックス .T は東証（日本株）のみ必要。
Qグラフを PNG ファイルとして保存したい
Plotly のグラフ右上のカメラアイコンをクリックすれば即 PNG 保存できる。コードで保存する場合は fig.write_image("chart.png")（!pip install kaleido が必要）。
