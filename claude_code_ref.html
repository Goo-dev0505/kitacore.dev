02. Service
サービス内容
Definition
ターミナル・IDE・デスクトップ・ブラウザで動くエージェント型コーディングツール

単なるコード補完ではなく、コードベース全体を理解した上でファイルの読み書き・コマンド実行・テスト・gitコミットまでを自律的に行うAIエージェント。CLAUDE.mdによる永続設定とMCP連携が他サービスとの本質的な差分。

03. vs Others
他社との違い
Core Difference
「チャット型」ではなく「エージェント型」

ChatGPTチャット / GeminiチャットがAIへの問い合わせ型であるのに対し、Claude Codeはコードベースをローカルで自律スキャンしファイルを直接読み書きして実行・テスト・コミットまで完結させる。CLAUDE.mdでプロジェクト知識を永続化し、毎回の説明コストをゼロにできる。なおChatGPTのCodex（クラウド型）も同様のエージェント機能を提供するが、ローカル実行・コンテキスト規模・MCP統合の深さで差がある。

2026年4月時点
※ Google Workspace データ読み書き 限定的　Claude Code = MCP設定要・読み書き可。ChatGPT = Team/Enterprise以上かつ読み込みのみ（書き込み不可）。
※ UI統合　GmailのUI内に「下書き作成」ボタンが表示・DocsのサイドバーでAIが編集・Sheetsでセル選択してそのまま分析、など各アプリのUI内にAIが直接組み込まれた状態。Claude CodeはMCP経由でデータ読み書きは可能だが、各アプリのUI内への組み込みはない。
※ 動画生成　ChatGPTのSoraはWebアプリ2026年4月26日・API2026年9月24日に終了予定。ChatGPTの画像生成はDALL-E / GPT-4o Imageが引き続き対応。Geminiの動画生成はVeo 3.1が担う。
※ スケジュール自動実行 限定的　ChatGPT・Geminiは定時プロンプト実行（通知で受け取る形）。Claude CodeのRoutinesはAnthropicサーバー上でコード実行・PRレビューなど開発タスクを直接動かす点で性質が異なる。
※ IDE拡張　VS Code・JetBrainsなどのコードエディタにインストールするプラグイン。エディタのUI内でdiff確認・指示出しができる。GeminiはGemini Code AssistでVS Code・JetBrains（IntelliJ/PyCharm/WebStorm）・Android Studioに正式対応。
※ MCP 限定的　ChatGPTは2025年9月のDeveloper Mode（ベータ）で読み書き対応。ただしリモートサーバーのみ・Plus/Pro必須。GeminiはAPI経由で対応。Claude Codeはローカル＋リモート両対応で最も整備されている。
※ シェル・テスト実行 限定的　ChatGPTはCodexのクラウドサンドボックス内で実行可能。Gemini CLIも端末コマンドに対応。いずれもローカル環境への直接アクセスではない。
※ 設定ファイル永続化　Claude Code = CLAUDE.md。ChatGPTのCodex = AGENTS.md（オープン標準・Cursor/Aider等でも共用可）。Geminiは同等の永続化ファイルは未整備。
※ エージェントチーム並行実行 限定的　CodexはGitHubリポジトリを複数クラウドコンテナで並列処理可能。ただしクラウド実行のみ・ローカル環境への直接アクセスなし。
※ Slack連携 限定的　CodexはSlack・GitHub・Linearと統合。ただしClaude Code（@メンションで直接PR作成）とは設定方法・統合の深さが異なる。
2026年4月時点の情報。各サービスの仕様は随時更新されます。
CLAUDE.md によるプロンプト削減
Before — 設定なし

「Jestを使って、コンポーネントは src/components に配置して、スタイルはTailwindで、MUIは使わないで、命名はcamelCaseで…」と毎回セッション冒頭で説明が必要。規約のズレも頻発する。

After — CLAUDE.md 設定済み

「このボタンにdark modeを追加して」

一言だけで完結。コーディング規約・技術スタック・禁止ライブラリはすべてセッション開始時に自動適用される。チームで共有すれば全員の指示が統一される。

04. Features
特徴の棚卸し
01
エージェント型アーキテクチャ
コードベースを最初に自律スキャンして全体構造を把握してから作業開始。ファイル間の依存関係・アーキテクチャを理解した上で複数ファイルを同時変更する。Unix哲学に従いパイプ・スクリプト・CI連携が可能。
02
CLAUDE.md による永続化
プロジェクトルートに置くだけでコーディング規約・アーキテクチャ決定・推奨ライブラリを永続化。/init でAIが自動生成。カスタムスラッシュコマンドでチーム共有ワークフローをパッケージ化できる。
03
大規模コンテキスト（200K〜1M tokens）
Sonnet 4.6 で200Kトークン（約15万単語）のコンテキストウィンドウ。大規模コードベース全体をメモリに保持したまま作業できる。GeminiのようなWebグラウンディングはないが、コード理解精度は高い。
04
Plan Mode（安全実行）
Shift+Tab で切り替え。ファイルに触れずに調査・計画だけを先行させる。大きな変更前に必ず使うことで誤操作を防げる。/rewind でチェックポイントへのロールバックも可能。
05
MCP（Model Context Protocol）
Notion・GCal・Google Drive・Slack等の外部サービスをエージェントが直接操作できるオープン標準。Anthropicが提唱しClaude Codeが最も積極的に対応している。
06
Hooks による自動化
PreToolUse / PostToolUse イベントでシェルコマンドを自動実行。ファイル編集後のPrettier自動実行・コミット前のlint実行・型チェック通過後のみ変更を受け入れるゲートなど。
07
エージェントチーム（サブエージェント）
複数のClaude Codeエージェントを並行起動。リードエージェントがサブタスクを割り振り・統合する。フロントエンド / バックエンド / テストを同時並行で進められる。
08
スケジュール実行（Routines）
Anthropic管理インフラで定期実行。PC電源オフでも動き続ける。毎朝PRレビュー・毎週依存関係監査・CI障害の夜間分析など定型タスクを全自動化できる。
CLIパイプ連携の例
bash
# ログの異常をSlackで自動通知 tail -200 app.log | claude -p "Slack me if you see any anomalies" # CIで新規文字列をフランス語に翻訳してPRを自動作成 claude -p "translate new strings into French and raise a PR for review" # セキュリティレビューを変更ファイル全体に適用 git diff main --name-only | claude -p "review these changed files for security issues"

05. Web Eval
ウェブでの評価
強みとして繰り返し挙げられる点
◎
コーディング品質：型安全・JSDoc付き・エッジケース対応まで行う。SWE-bench 82.1%でトップクラス
◎
文章品質・指示への忠実さ：複数レビューで「最もよく挙げられる強み」。自然なリズム・段落構造・トーン指示への追従
◎
長文コンテキスト処理：非常に長いドキュメントの深い部分から特定詳細を取り出す精度が高い
◎
ハルシネーション率の低さ：独立テストで約3%。ChatGPT / Gemini の約6%と比較して最低水準
◎
開発者エコシステムの中心：Cursor・Windsurfがバックエンドに採用。Google / Microsoft / OpenAI社員も業務利用
弱みとして挙げられる点
△
マルチモーダル（動画・音声）：動画・音声処理・動画生成はGeminiが明確に優勢。Claude自体は画像入力に対応するが、動画・音声はネイティブ非対応
△
Google Workspace 連携：G Suite中心の組織にはGeminiの方が大幅に有利
△
APIコスト：Gemini Flash等と比較してAPI単価が高め。大量処理には要検討
△
リアルタイム情報：Webグラウンディングの強さではGemini・Grokに劣る
△
完全委任の限界：Anthropic調査でも完全委任できるタスクは0〜20%。残りは人間のレビューが必要

エンジニアが業務の60%にClaudeを活用し、生産性50%向上を報告。完全委任できるのは0〜20%。残りは人間のレビューが必要。

TDDとの相性

テスト駆動開発（TDD）を実践しているチームが最も恩恵を受けるという評価が定着。テストのないプロジェクトでは品質管理が難しい。

企業採用状況

Microsoft・Googleが主要エンジニアリングチームで内部採用。Fortune 10のうち8社がClaudeを利用（Anthropic発表）。
